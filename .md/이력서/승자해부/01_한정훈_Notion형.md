### 프로젝트 1 — WinAPI 익스트랙션 슈터 멀티플레이 (C++)
- 헤더 블록: 게임 설명(익스트랙션 슈터/RPG, 랜덤 맵, 파밍·생존·탈출) / **1인 개발(클라+서버) / 2024.01.26~03.01 (5주) / 목적 명시 / 기술: C++, winapi**
- 항목 구성 (7개):
  1. **IOCP 서버 프레임워크** — 비동기 소켓 + 멀티스레드
  2. **WinAPI 클라이언트** — "상용 엔진을 사용하다가 작동 원리를 파악하기 위해 로우 레벨에서 직접 구현" 명시
  3. **디버깅 툴 개발** — 스프라이트/애니메이션 툴, 맵 에디터, 애니메이션 에디터 (이미지)
  4. 서버 랜덤 맵 생성
  5. **2D 타일 이동 동기화 + 충돌**: 매 타일 이동 패킷 → 패킷 폭증 문제 → **방향 전환/점프/멈춤 시에만 전송** + RTT/2 딜레이 적용 + 타 클라 위치·상태 롤백 보정
  6. **MySQL DB 연동** — Connection 잦은 생성/삭제 문제 → **DB Connection Pool**
  7. **Job Queue로 I/O 싱글스레드 관리** — Lock 멀티스레드의 DeadLock 회피 → GameRoom=JobQueue, DoAsync로 큐 적재, 처리(이동/공격)는 싱글스레드 → **MPSC lock-free 큐**로 효율화. 코드 일부 노출 (ServerPacketHandler)
- 게임 플레이 영상: **좌측 클라이언트 양측 분할 + 서버 cmd 동시 촬영 + 자막** (유튜브)

### 프로젝트 2 — 언리얼 MMORPG 서버 (C++, 협업)
- 헤더: **2인(클라 1 + 서버 1=본인) / 2024.07.19~09.02 / 기술: C++, UE, google protobuf** — "협업 어필 목적의 팀 구성" 명시
- 항목:
  1. **UE 데디케이티드 서버를 쓰지 않고 자체 IOCP 서버** — 3D 컨텐츠에 맞는 서버 구현 학습 목적
  2. 언리얼 네트워크 프레임워크 개발
  4. **클라-서버 맵 데이터 연동**: 맵을 2D 격자로 분리, LineTrace로 격자별 height 저장, 액터 설정값 기반 8방향 이동 정의 (파란=가능/빨강·노랑=불가 시각화)
  5. **QuadTree 충돌 + AOI**: 공간 분할로 주변 공간만 충돌 검사(돌진 스킬·AI 이동), AOI로 주변 유저만 이동 동기화 → 패킷 수+클라 부담 감소
  6. **논타겟팅 스킬 시스템**: 스킬 데이터 추상화 + JSON 서버 저장, SkillActor 월드 액터로 충돌 처리. 차징/홀딩/캐스팅 + 피해/버프/디버프/도트 구분
  7. **보스/적 AI**: A* 길찾기, 서버가 애니·공격 타이밍 조절해 클라 동기화

## 분석

### 장점 (훔칠 것)
- **문제→해결 서사가 항목 단위로 반복** (패킷 폭증→이벤트 기반 전송, DeadLock→JobQueue+MPSC) — 모든 기술 항목이 스토리
- **"상용 엔진 원리를 알려고 로우레벨 직접 구현" 프레이밍** — WinAPI 프로젝트를 '낡은 기술'이 아니라 '본질 이해'로 승격
- **자체 서버 선택의 이유 명시** — UE 데디 서버 대비 왜/무엇을 배우려고
- 영상 규격: **클라 2분할 + 서버 cmd + 자막** — 서버 동기화의 시각적 증명
- 헤더 블록 규격: 인원/기간/목적/기술 4종 고정

### 단점 (배울 것)
- 수치가 거의 없음 (프레임/패킷 수 비교 부재) — 경력 보정 추정("포폴이 그렇게 좋지는 않은데 경력빨?")
