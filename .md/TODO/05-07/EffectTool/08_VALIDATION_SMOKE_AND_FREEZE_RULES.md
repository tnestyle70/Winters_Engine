## 2. RHI boundary grep

FX v2 public/private에서 native API를 직접 쓰면 실패다.

```powershell
rg "ID3D11|ID3D12|CDX11Device|RHI/DX11|RHI/DX12" Engine/Public/FX/v2 Engine/Private/FX/v2
rg "GetNativeHandle" Engine/Public/FX/v2 Engine/Private/FX/v2
```

기대:

```txt
EFX-0/1/2
  Engine/Public/FX/v2 native hit 0.
  Engine/Private/FX/v2 native hit 0.

EFX-3 이후
  FX renderer도 IRHICommandList와 RHI handles만 사용.
```

예외:

```txt
Engine/Private/RHI/DX11
Engine/Private/RHI/DX12
기존 legacy Client FX files
```

## 10. Freeze 규칙

EFX-0 freeze:

```txt
No legacy callsite without manifest entry.
No absolute asset path.
No direct removal of legacy spawn fallback.
```

EFX-1 freeze:

```txt
No manual string parser in FX/v2.
No non-canonical .wfx/.wmi generated by tool.
No schema field without load/save/round-trip test.
```

EFX-2 freeze:

```txt
No runtime object ownership inside ECS POD component.
No render World query in FX renderer.
No same-phase producer/consumer systems.
```

EFX-3 freeze:

```txt
No native DX11/DX12 include in Engine/Public/FX/v2.
No DX12 visual pass with no-op command bindings.
No shader register without space0.
No PBR dependency in VFX master material.
```

EFX-4 freeze:

```txt
No champion-specific function call from generic EffectToolPanel.
No clipboard-only save as primary workflow.
No preview asset with unsaved hidden state.
```

EFX-5 freeze:

```txt
No GPU feature marked complete without DX12 path.
No CollisionQuery fake success.
No compute/render resource state ambiguity.
```
