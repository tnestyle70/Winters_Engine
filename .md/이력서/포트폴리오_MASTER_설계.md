# 포트폴리오 MASTER 설계 — 2400++ (승자 3인 해부 통합)

> 형식 결정: **Notion** (사용자 확정 — "하이라이트 영상이 바로 보이는" 접근성이 결정 근거)
> 원칙: 승자 3인의 장점 전부 + 단점의 반대 실행. 상세 내용은 사용자가 작성 중 — 이 문서는 양식·배치·본질.

## 0. 이력서/포트폴리오의 본질

**채용의 본질 = 리스크 구매.** 회사는 "이 사람을 뽑았을 때 실패할 확률"을 산다. 그래서 포트폴리오의 본질은 자랑이 아니라 **리스크 감소 증거의 체계**다. 면접관의 3단 소비 구조에 맞춰 설계한다:

1. **30초 스캔** → 목차·요약·영상 썸네일이 답한다 ("이 사람 뭐 하는 사람인가")
2. **관심 주제 딥다이브** → 문제 정의→접근→선택 이유→수치 결과가 답한다 ("문제를 어떻게 다루는 사람인가")
3. **역질문 소재 수집** → trade-off 서술과 '한계 및 개선점'이 답한다 ("자기가 뭘 모르는지 아는가")

**문제의 정의·설정·발견을 녹이는 방법** (사용자 질문에 대한 답): 모든 항목을 아래 5칸 구조로 강제한다 — 이게 세 승자의 공통 골격이고, 특히 김태호형의 힘이다.

```text
[의도] 왜 이걸 만들려 했나 (궁금증/필요)
[1차 접근과 문제 발견] 해봤더니 이런 문제가 — 문제를 수치로 서술 (주관 표현 금지)
[해석] 그 문제가 왜 생기는지 CS 지식으로 설명
[해결과 선택 이유] 왜 이 방법인가, 대안 대비 trade-off
[결과] Before/After 수치 표 + 한계 및 개선점
```

## 2. 전체 구조 (Notion 페이지 트리)

```text
[홈] 이름 — Game Server/Client Programmer
├─ 정체성 3줄 (한정훈식: 관심사·문제 접근 태도·방향성)
├─ ★ 2분 하이라이트 영상 (임베드, 즉시 재생)
├─ 대표 수치 5개 (17.8ms→9ms / 300틱 해시 / 커밋 64%·PR 135 / Steam 출시 / 챔피언 15종 30Hz)
├─ 목차 (아래 순서 그대로, 읽는 시간 표기)
│
├─ 1. [딥다이브] Winters — 자체 엔진 서버 권위 멀티플레이 (메인, 김태호형)
├─ 2. Minecraft Dungeons — 5인 팀 협업 + IOCP 서버 (한정훈형)
├─ 3. Starcraft — WinAPI 로우레벨 RTS 코어 (한정훈형 압축)
├─ 4. NYPC — 게임 AI 대회 (MCTS/self-play 하네스) (PPT형 압축)
├─ 5. Liberation — Steam 상용 출시 (서사 카드)
├─ 6. 기타 이력 (PPT형)
└─ 자기소개 서사 (문과→Liberation→아카데미→팀협업→자체 엔진, 회고+선택 동시 제시)
```

순서 논리: **깊이(1) → 협업(2) → 기본기(3) → 지능(4) → 완성(5)**. 면접관이 1번만 읽어도 채용 리스크의 대부분이 해소되게.

## 3. 메인 딥다이브 설계 — Winters (김태호형 5칸 적용)

- **[의도]** LoL급 게임의 자체 엔진들(라이엇/기존 기업)처럼, UE/Unity와 객관 비교해 경쟁력 있는 엔진을 목표로 설계 (사용자 서사)
- **[문제 정의]** 멀티플레이 MOBA의 본질 문제 = 치팅 방지 + 동기화 + 결정성 → 서버 권위 구조 선택
- **["왜 UE 데디 서버가 아닌가"]** (한정훈 확장, 사용자 계획): 데디 서버의 제약(틱 제어·relevancy 커스터마이즈·운영 비용) → 자체 30Hz 틱·커맨드 검증·시야 기반 스냅샷 필터 → 실운영 확장에 필요한 것(버전 핸드셰이크·크래시 리포팅 — 감사로 파악 완료)까지 언급 = "운영을 생각해본 사람" 신호
- **[해결 구조]** 5단 파이프라인 + GameCommandWire/GameCommand 분리 + 봇도 커맨드 생산자 + Lag Compensation + 예측 보호(12스냅샷 한도)
- **[결과 수치 — 확보됨]** 17.8ms→9ms(프로파일러 계측) / 300틱 결정성 해시 무회귀 / FBX 27종 FAIL 0 / 60MB→1.2MB
- **[결과 수치 — 추가 계측하면 김태호식 표 완성]** FOW 스냅샷 필터 Before/After (엔티티 수·바이트/틱), 커맨드 코얼레싱 전후 패킷 수
- **[한계 및 개선점 (정직)]** JobSystem: Chase-Lev race 미해결로 비활성 / UDP 미이관(TCP) / 에디터 원자적 저장 부재 — 각각 "다음 단계"로 서술
- **[예상 역질문 준비]** "서버 권위인데 반응성은?"(예측 보호 trade-off), "30Hz 근거는?", "FlatBuffers 선택 이유는?(protobuf 대비)"

### ⚠️ FAULT 금지 슬롯 (기재 보류 — 구현·계측 후에만)
- **Fiber** (커널 전환 오버헤드, 너티독 GDC 서사): 6주 마스터리 미착수 → 구현+전후 수치 확보 전 기재 금지
- **JobSystem/Chase-Lev/Work-Stealing**: 구현은 존재하나 race로 비활성 → "구현했고 켜져 있다"로 읽히는 서술 금지. 넣으려면 (a) race 수정+활성화+계측 or (b) 한계 섹션에 정직 기재 중 택1
- 이 둘을 넣고 싶은 욕구 = 완벽주의 바닥 신호가 아니라 **정당한 천장 작업 후보** — 단, 제출을 막지 않는 선에서 (제출 후 v2 업데이트로)
