# Ch10. UI (UMG-tier / DataBinding / Localization / Slate)

> Winters 현재: `Client/Public/UI/` 씬별 UI, `Engine/Public/Renderer/UIRenderer.h` 단일 renderer.
> 레퍼런스: `UnrealEngine/Engine/Source/Runtime/UMG/Public/`, `Runtime/SlateCore/Public/`, `Runtime/Slate/Public/`.

---

## 1. 기초 원리 — UI는 또 하나의 렌더 시스템이다

게임 UI에 필요한 것:

1. **Layout** — 화면 어디에 어떻게 배치 (반응형, 다양한 해상도/aspect)
2. **Render** — text/image/shape를 GPU로 효율 그리기 (보통 batched 2D)
3. **Input** — 마우스/터치/패드 fokus + event bubbling
4. **State binding** — gameplay 상태 → UI 자동 갱신 (HP가 80 → 바도 80%)
5. **Animation** — UI 자체 애니메이션 (메뉴 슬라이드, 데미지 popup)
6. **Localization** — 다국어 + 가독성 (한자/아랍 RTL/이모지)
7. **Theming** — 모든 위젯에 일관된 디자인

옛날 게임은 1번을 그냥 코드로 픽셀 좌표 박았다. AAA는 이걸 **데이터 + retained mode** 시스템으로 푼다.

## 2. 핵심 — UE5 Slate / UMG 6대 구성

### 2.1 SWidget (Slate, 저수준)

`Source/Runtime/SlateCore/Public/Widgets/SWidget.h`. 모든 위젯의 base.

immediate vs retained:
- **Immediate** (Dear ImGui): 매 프레임 `ImGui::Button("OK")` 호출. 상태가 호출 코드 측에.
- **Retained** (Slate): 위젯 객체 트리를 만들어 유지. 매 프레임 paint만.

Slate는 retained. UE5 게임 UI / Editor UI 둘 다 SWidget 기반.

```cpp
class SWidget
{
public:
    virtual void Tick(const FGeometry& AllottedGeometry, double InCurrentTime, float InDeltaTime);
    virtual int32 OnPaint(const FPaintArgs& Args, const FGeometry& AllottedGeometry,
                          const FSlateRect& MyCullingRect, FSlateWindowElementList& OutDrawElements,
                          int32 LayerId, const FWidgetStyle& InWidgetStyle, bool bParentEnabled) const;

    // Layout
    virtual FVector2D ComputeDesiredSize(float) const = 0;
    virtual FChildren* GetChildren() = 0;

    // Input
    virtual FReply OnMouseButtonDown(const FGeometry& Geom, const FPointerEvent& Event);
    virtual FReply OnKeyDown(const FGeometry& Geom, const FKeyEvent& Event);
};
```

### 2.2 Layout (Slot-based)

위젯에 자식을 추가할 때 **slot** 통해. slot이 layout 메타데이터(alignment, padding, fill)를 들고 있음.

```cpp
SVerticalBox::FSlot* NewSlot = nullptr;
SAssignNew(MyContainer, SVerticalBox)
+ SVerticalBox::Slot()
  .AutoHeight()
  .Padding(10)
  [
      SNew(STextBlock).Text(FText::FromString("Health"))
  ]
+ SVerticalBox::Slot()
  .HAlign(HAlign_Fill)
  [
      SNew(SProgressBar).Percent(this, &MyWidget::GetHealthPercent)
  ];
```

레이아웃 위젯: SHorizontalBox, SVerticalBox, SGridPanel, SOverlay, SUniformGridPanel, SCanvas, SSplitter.

### 2.3 UMG (User Motion Graphics, 고수준)

`Source/Runtime/UMG/Public/Components/Widget.h`.

UMG = Slate 위에 **UObject로 감싼 위젯** + **Blueprint 편집기**. 게임 UI 1차 선택지.

```cpp
class UWidget : public UVisual
{
public:
    UFUNCTION(BlueprintCallable, Category="Visibility")
    void SetVisibility(ESlateVisibility InVisibility);

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Appearance")
    FWidgetTransform RenderTransform;

    UPROPERTY(EditAnywhere, BlueprintReadWrite, Category="Behavior")
    bool bIsEnabled;

protected:
    // Slate 위젯 생성
    virtual TSharedRef<SWidget> RebuildWidget();
};
```

디자이너 워크플로:
1. WBP_MainHUD UMG 에디터에서 위젯 배치 (드래그)
2. ProgressBar, Image, TextBlock 등 추가
3. property를 binding (HP 값 → ProgressBar.Percent)
4. Blueprint 그래프로 click handler

### 2.4 Property Binding

UI는 매 프레임 gameplay 상태를 polling하거나 push notification으로 갱신.

```cpp
// Pull (Slate Attribute)
SNew(SProgressBar)
    .Percent_Lambda([this]() { return Character->HealthPct(); });   // 매 paint마다 호출

// Push (UMG Property Binding)
WBP_MainHUD ─ delegate bind ── ICharacter::OnHealthChanged ── Set ProgressBar.Percent
```

UE5 5.x **MVVM** (Model-View-ViewModel) 플러그인이 push 방식 권장.

### 2.5 Animation

`UnrealEngine/Engine/Source/Runtime/UMG/Public/Animation/WidgetAnimation.h`.

WidgetAnimation = sequencer 트랙. transform/color/opacity 등을 시간축에 따라 변경.

```cpp
class UUMGSequencePlayer
{
    void Play(float StartAtTime, int32 NumberOfLoops, EUMGSequencePlayMode::Type PlayMode, float PlaybackSpeed);
};
```

데미지 popup, 메뉴 슬라이드, 알림 fade in/out 모두 widget animation.

### 2.6 Localization

```text
LOCTEXT("MainMenu_PlayButton", "Play")
// → 빌드 시 .po 파일 추출
// → 번역가가 ko/ja/zh 채움
// → 런타임 culture 변경 시 자동 swap
```

UE5: `Internationalization.h`, `TextLocalizationManager.h`.

문제 케이스:
- 한국어/일본어 폰트 (CJK glyph는 수만 개 → atlas 못 만듦, distance field 또는 dynamic atlas)
- 아랍어 RTL (오른쪽→왼쪽)
- 동남아 (Thai/Vietnamese — diacritic 복잡)
- 가독성 차이 (영어 텍스트 길이 ≠ 독일어)

## 3. 심화

### 3.1 Retained Mode + Fast Path

UE5 5.x **Invalidation Box**: 위젯 트리에서 안 바뀐 부분은 paint 안 함. dirty propagation으로 효율.

```cpp
class SInvalidationPanel : public SCompoundWidget
{
    // 자식이 dirty 마킹할 때까지 cached geometry 재사용
};
```

수백 위젯 화면에서도 60fps 유지.

### 3.2 Material in UI

UI 위젯도 material 사용 가능. 셰이더 effect (블러, 글로우, 그라데이션 노이즈). HUD 디자이너의 강력한 도구.

### 3.3 Common UI Plugin

UE5 5.x `CommonUI`: 콘솔/PC/모바일 입력 통합, focus navigation, action bar, activatable widget stack.

LoL/로아 같은 게임이 PC + 모바일 동시 가는 경우 필수.

### 3.4 World-space UI

화면 좌표 UI 외에 3D world에 떠 있는 UI (HP바, NPC 이름표).

```cpp
class UWidgetComponent : public UMeshComponent
{
    UPROPERTY(EditAnywhere)
    EWidgetSpace Space;   // Screen / World
};
```

캐릭터에 attach → 3D 변환 → 카메라 facing.

### 3.5 Accessibility

color blind mode, font size scaling, screen reader, controller-only nav. AAA 라이브 서비스 필수.

UE5: `Source/Runtime/SlateCore/Public/Widgets/Accessibility/`.

### 3.6 Performance

UI는 의외로 비싸다.
- 위젯 100개 = layout 100회 + paint 100회
- text는 글리프별로 quad → 1000자 = 1000 quad
- material UI는 fillrate 폭주

해결책: InvalidationPanel, retained mode, atlas pre-bake, font SDF.

// Engine/Public/UI/Widget.h
class WINTERS_ENGINE CWidget
{
public:
    virtual ~CWidget() = default;
    virtual Vec2 ComputeDesiredSize() const = 0;

    virtual void Paint(UIDrawList& out, const UIGeometry& geom) const = 0;

    virtual eUIEventResult OnMouseDown(const UIMouseEvent& e) { return eUIEventResult::Unhandled; }

    void SetVisibility(eUIVisibility v);

### 4.3 Bot AI 무관성

Ch10은 클라 전용. 서버는 UI를 모른다. Bot AI도 UI에 접근하지 않는다.

### 4.4 단계별

```text
Ch10-Stage1  CWidget + Panel + Primitive 기본 (HVBox, Text, Image, Button)
Ch10-Stage2  Layout 패스 (ComputeDesiredSize → arrange children)
Ch10-Stage3  Invalidation (dirty propagation)
Ch10-Stage4  Property binding (TBindable)
Ch10-Stage5  WidgetAnimation (transform/color/opacity track)
Ch10-Stage6  Material UI (셰이더 effect)
Ch10-Stage7  Localization (CJK font SDF, RTL)
Ch10-Stage8  World-space UI (캐릭 HP바)
Ch10-Stage9  Accessibility / Common UI (PC + 모바일 입력 통합)
Ch10-Stage10 UMG-tier 에디터 (Ch12와 묶임)
```
