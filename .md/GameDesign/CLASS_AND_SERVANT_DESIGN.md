# Class & Servant — 핵심 게임 디자인 문서

> **"전략 게임의 새로운 차원을 여는 MOBA"**
> 스토리·연출이 아닌 **결정의 압력** 자체를 디자인의 중심에 둔다.
> LoL의 공간·자원·롤 시스템과 엘든링의 빌드·숙달·카타르시스를 결합하되,
> 두 게임 어느 쪽도 가지 못한 **Dual Agency**라는 새 좌표를 연다.

---

## 이 문서의 구조

```
Phase 0  전략 게임 재미의 20층위 (Strategic Fun Atlas)
Phase 1  레퍼런스 게임의 Fun Fingerprint
Phase 2  Class & Servant의 정체성 좌표
Phase 3  Dual Agency를 게임 메커닉으로 구체화
Phase 4  맵 구조 — PvE-PvP 하이브리드의 좌표
Phase 5  의도적으로 안 하는 것 (스코프 방어)
Phase 6  다음 결정점
```

# Phase 0 — 전략 게임 재미의 20층위 (Strategic Fun Atlas)

**전제**: 폴가이즈, 어몽어스, 포커, 장기, 오목, 배틀그라운드, 에이펙스, 배틀필드, 철권, 스파, 하스스톤, LoL, 엘든링 PvP — 이 모든 게임의 공통 본질은:

> **"불완전한 정보 위에서 인간이 인간에게 결정으로 압력을 가하는 행위"**

스토리·그래픽·사운드는 그 위에 입혀진 옷이고, 본체는 결정 흐름이다. 그래서 20층위는 **"어디서 결정의 압력이 발생하는가"** 의 축으로 분류한다.

| # | 층위 (Layer) | 본질 | 이 층위가 지배적인 게임 |
|---|---|---|---|
| 1 | **결정의 무게** | 한 결정이 결과에 미치는 비선형 영향 | 포커 올인, LoL 정글 갱 |
| 2 | **불완전 정보** | 안개·핸드·정체 — "모름"이 만드는 긴장 | 어몽어스, 포커, LoL Fog |
| 3 | **읽기 / 속임수** | 상대 의도 추론 + 역이용 | 격투, 포커, 구스구스덕 |
| 4 | **공간 지배** | 위치 선점이 곧 가치 | 장기, 배그 자기장, LoL 비전 |
| 5 | **자원 비선형성** | 작은 차이 → 큰 격차 (스노우볼) | LoL 골드, 하스스톤 마나 |
| 6 | **시간 압박** | 클럭이 결정 질을 압축 | 격투 프레임, 배그 zone, 블리츠 체스 |
| 7 | **타이밍 / 콤보** | 입력의 정확한 결합 | 철권, 스파, 액션 |
| 8 | **롤 비대칭** | 같은 목표, 다른 능력 | 어몽어스(임포/크루), MOBA 롤 |
| 9 | **사회적 신뢰·배신** | 인간 vs 인간의 메타게임 | 어몽어스, 구스구스덕 |
| 10 | **숙달 곡선** | 시간 → 실력 환원율 | 격투, MOBA, 엘든링 빌드 |
| 11 | **빌드 / 적응** | 게임 중 자기 키트를 만들어감 | LoL 아이템, 하스스톤 멀리건, 엘든링 |
| 12 | **창발** | 단순 규칙에서 복잡한 상황 | 모든 우수한 전략 게임 |
| 13 | **카타르시스** | 절체절명 → 역전 / 생존 | PvP 절정, 엘든링 보스 막타 |
| 14 | **복수 동기** | 진 직후 다시 하고 싶은 충동 | 엘든링, 격투, 오목 |
| 15 | **공유 서사** | "그때 그 판"의 회고 가능성 | 어몽어스 토론, MOBA 하이라이트 |
| 16 | **확률 다스림** | 운을 EV로 환원하는 능력 | 포커, 하스스톤 |
| 17 | **자기표현** | 픽 / 스타일이 자아 표현 | LoL 챔프, 격투 캐릭, 엘든링 빌드 |
| 18 | **소속감** | 팀·메타·길드의 일원 감각 | MOBA 5v5, 길드 |
| 19 | **인지부하의 쾌감** | 머리 굴리는 행위 자체의 즐거움 | 장기, 포커, RTS |
| 20 | **물리적 숙달** | 손가락의 정밀성 | 격투, FPS, 액션 |

**이 격자가 의미 있는 이유**:
모든 위대한 게임은 20층위 중 **3~5개를 비정상적 깊이로 파고**, 나머지는 평균만 친다. 모두 평균이면 평범하고, 한두 개에만 올인하면 협소하다. 이 게임이 어디를 깊이 팔지가 곧 정체성이다.

# Phase 1 — 레퍼런스 게임의 Fun Fingerprint

각 게임을 20층위에 점수 매겨 보면 **어디가 비어 있는지**가 보인다:

```
LoL          : [4,5,11,17,18] 깊음 — 공간/자원/빌드/롤/팀
              [3,9,16] 약함     — 읽기/사회배신/확률

엘든링       : [10,11,13,14,17] 깊음 — 숙달/빌드/카타르시스/복수/표현
              [2,3,9,18] 약함    — 정보전/심리전/사회/소속 (PvP 빼면)

어몽어스     : [2,3,9,15] 극단적 깊음 — 정보/읽기/배신/서사
              [7,11,17,20] 약함  — 콤보/빌드/표현/물리숙달

배그/에이펙스: [4,6,16] 깊음     — 공간/시간/확률
              [11,15,18] 부분    — 빌드는 얕고 서사는 우연

포커         : [1,2,3,16,19] 핵심 — 결정무게/정보/읽기/확률/인지
철권/스파    : [3,6,7,10,20] 핵심 — 읽기/시간/콤보/숙달/물리
하스스톤     : [5,11,16,19] 핵심  — 자원/빌드/확률/인지
장기/오목    : [4,12,19] 핵심     — 공간/창발/인지
폴가이즈     : [13,15,20] 핵심    — 카타르시스/서사/물리 (얕은 깊이)
```

**비어 있는 조합** — 아무도 깊게 안 판 영역:

- **[3 읽기 × 4 공간 × 11 빌드]**
  → 격투의 심리전을 MOBA 공간에서, 빌드를 게임 중 적응형으로

- **[2 정보 × 8 롤비대칭 × 18 소속감]**
  → 어몽어스의 정보 비대칭을 5v5 팀 컨텍스트로 (롤 자체가 정보 비대칭)

- **[10 숙달 × 13 카타르시스 × 14 복수]**
  → 엘든링의 "패배가 곧 학습" 사이클을 PvP에 (소울즈 침입과 가까움)

이 셋의 교집합이 **Class & Servant의 "새 차원"** 이 들어갈 자리다.

# Phase 2 — Class & Servant의 정체성 좌표

이름부터 분석한다.
- **"Class"** = 클래식 MOBA 챔피언, 정체성·롤
- **"Servant"** = 종속적 무엇 — 펫? 분신? 그림자? 소환물?

여기서 가장 중요한 디자인 선택이 들어간다.

## 핵심 디자인 무브 (Core Design Move)

> **Dual Agency** — 한 명의 플레이어가 "Class (주)"와 "Servant (부)"라는
> **이질적 메커닉을 가진 두 유닛**을 동시에 운용한다.
> 전환은 자유롭지만, 한 쪽을 조작하는 동안 다른 쪽은 **약화된 자율 행동(Auto)** 만 한다.

## 왜 이게 새로운 차원을 여는가

### 1. 인지부하의 새 축 (층위 19)

```
MOBA  : 유닛 1개를 깊이 운용
RTS   : 다수 유닛을 얕게 운용
─────────────────────────────
빈 공간 : 2개 유닛, 각자 깊이
```

이 빈 공간은 솔로큐 PvP에서 거의 안 가본 길이다. Dota Meepo, LoL Tibbers는 **예외적 챔프**지 게임의 본질이 아니었다.

### 2. 롤 비대칭의 게임 내부화 (층위 8)

어몽어스가 "임포 vs 크루"로 비대칭을 만든 걸, **한 플레이어 안에서 비대칭**을 만든다.
너의 Class는 전사인데 너의 Servant는 정찰 / 함정 / 소환물일 수 있다.
**자기 안의 두 역할이 서로를 보완**하는 구조.

### 3. 숙달 곡선의 재정의 (층위 10)

```
기존 MOBA 마스터리:
  스킬 4개 + 이속 = 마스터리

Class & Servant 마스터리:
  Class 키트
  + Servant 키트
  + 둘의 시너지
  + 컨텍스트 전환 타이밍
  = 마스터리
```

엘든링 빌드 깊이를 PvP 솔로 컨텍스트에 이식.

### 4. 공간 지배의 분할 (층위 4)

Class가 미드라인을 잡는 동안 Servant는 정글을 도는 게 한 명으로 가능.
**혼자서 두 지점에 영향력을 행사**한다.
LoL의 "정글-라이너 분리"를 한 명의 결정으로 압축.

### 깊이가 박히는 층위

이 무브 하나가 **[3, 4, 8, 10, 11, 19]** 여섯 층위에 동시에 깊이를 박는다. 나머지 층위는 표준적인 MOBA로 두면 된다.

# Phase 3 — Dual Agency를 게임 메커닉으로 구체화

추상 개념을 실제 결정 압력으로 바꾸려면 4개의 서브시스템이 필요하다.

## (A) Possession 시스템 — "지금 누구의 시점인가"

- **기본 상태**: 카메라·입력은 Class. Servant는 AI 오토.
- `Tab` (또는 `Alt`+클릭)으로 Servant로 빙의. 이 순간 Class가 AI 오토.
- **AI 오토는 의도적으로 약함** — 추격은 안 하고 후퇴/방어만 함.
  → "내가 안 잡고 있는 유닛은 손실 중"
  → 이 압박이 곧 결정의 무게(층위 1)
- 빙의 전환에는 **0.5초 글로벌 쿨**이 있어 — 패닉 스왑으로 두 마리 다 살 수는 없음.
  → **포기를 결정으로 만든다.**

## (B) Bond 시스템 — "둘의 거리가 곧 관계"

Class와 Servant 사이 거리에 따라 양쪽 모두에 버프/디버프:

| 거리 상태 | 효과 | 의미 |
|---|---|---|
| **Bound (근거리)** | 둘이 함께 강함. 시너지 발동. | 모여서 한 점 압박 |
| **Stretched (중거리)** | 표준. 시너지 없음. | 중립 상태 |
| **Severed (원거리)** | 둘 다 약화. **그러나 영향력의 면적은 두 배.** | 흩어져서 두 곳 압박 |

**이게 만드는 결정**: "지금 모여서 강해질까, 흩어져서 두 곳을 압박할까."
→ LoL의 "그루핑 vs 스플릿푸시" 결정을 **한 플레이어 머릿속**으로 옮긴 것.

## (C) Asymmetric Kit — "둘은 서로를 닮으면 안 된다"

**디자인 원칙**: Class와 Servant의 키트는 **반드시 다른 자원·다른 사거리·다른 시간 스케일**을 써야 한다.

```
예시:
  Class    : 마나 기반 즉발 근접
  Servant  : 쿨다운 기반 원거리 설치형
```

닮으면 그냥 "두 마리 같은 유닛"이 되어 인지부하만 있고 **결정의 차별성은 0**.
이게 깨지면 게임 본질이 무너진다.

### 데모 페어 후보

2개 데모 클래스로는 두 가지 극단 페어를 검증해야 한다:

- **Pair A: 즉발-반응형 (Class) × 설치-예측형 (Servant)**
  → 격투 심리전 결을 가져옴

- **Pair B: 이속-스킬샷형 (Class) × 저속-탱커형 (Servant)**
  → 미끼/안전망 결

## (D) Servant Death — "둘 중 하나가 죽으면"

- 둘 다 죽어야 **사망 카운트 1**. 그러나 하나만 죽어도 페널티는 큼.
- **Servant 죽음** → Class에 영구 디버프(이속 -10%) 30초간. Bond 시스템이 무너졌단 신호.
- **Class 죽음** → Servant는 살아 있어도 다음 리스폰까지 **자기 둥지로 후퇴만 가능** (조작 불가).

**이게 만드는 결정**: "Servant를 미끼로 던질 가치가 있는가."
→ 엘든링의 "Spirit Ash 소환물 미끼 전술"의 PvP화
→ [13 카타르시스 × 14 복수]의 진입로

# Phase 4 — 맵 구조: PvE-PvP 하이브리드의 좌표

엘든링의 "방대한 맵 + 보스"를 PvP에 쑤셔 넣는 데는 함정이 있다.
**PvE 시간이 PvP 결정 압력을 희석**한다.

그래서 엘든링 모작이 아니라, 엘든링의 **"맵이 곧 자원이고 정보다"** 라는 본질만 가져와야 한다.

## 삼중원 맵 (Triple-Ring Map)

```
        ╔═══════════════════════════════════╗
        ║  Outer Ring — Wilderness          ║
        ║  ├ 미니 보스 3마리 (각 라인 끝)   ║
        ║  ├ 강력하지만 시간 비용 큰 드랍   ║
        ║  └ Servant만 빠르게 통과 가능     ║
        ║                                   ║
        ║   ╔═══════════════════════════╗   ║
        ║   ║  Middle Ring — Battle     ║   ║
        ║   ║  ├ 표준 MOBA 라인 3개     ║   ║
        ║   ║  ├ 미니언/포탑/정글 캠프  ║   ║
        ║   ║  └ Class·Servant 자유     ║   ║
        ║   ║                           ║   ║
        ║   ║   ╔═══════════════════╗   ║   ║
        ║   ║   ║  Inner Sanctum    ║   ║   ║
        ║   ║   ║  ├ 본진 + 코어    ║   ║   ║
        ║   ║   ║  ├ Servant 풀링   ║   ║   ║
        ║   ║   ║  └ 그레이스 타워  ║   ║   ║
        ║   ║   ╚═══════════════════╝   ║   ║
        ║   ╚═══════════════════════════╝   ║
        ╚═══════════════════════════════════╝
```

**핵심**:
**Outer Ring은 Servant 전용 PvE 영역**이다. Class는 못 들어가거나 들어가도 매우 느리다.

게임 초반의 결정이 갈린다:
- "내가 Class 잡고 라인전, Servant 풀어서 Outer Ring PvE"
- "둘 다 라인 압박"

이게 **레인전과 정글 결정을 한 사람 안에서 일어나게** 만드는 구조.
LoL의 정글러 메타가 만든 게임 맥동을 **모든 플레이어가 동시에 경험**하는 셈.

# Phase 5 — 의도적으로 안 하는 것 (스코프 방어)

천재 디자이너의 절반은 **빼는 능력**이다.

## 우리가 안 하는 것

- ❌ **AAA 비주얼 / 시네마틱**
  → 스토리·연출에 시간을 쓰지 않는다. 본질 외 비용.

- ❌ **거대한 챔프 풀**
  → 데모는 Class 2개 × Servant 2개 = 4페어 고정. 메타 검증 후 확장.

- ❌ **랜덤 박스 / 가챠**
  → [16 확률 다스림] 층위는 게임 메커닉 안에 두지, 수익화에 두지 않음.

- ❌ **싱글 캠페인**
  → PvP 정체성을 흐림. 후속작에서 가능.

- ❌ **얕은 사회 시스템**
  → 길드·채팅은 핵심 루프 검증 후 (층위 18은 후순위).

## 데모 v0.1 정의 (협상 불가)

```
✅ Class 2개 × Servant 2개 = 4페어
✅ 삼중원 맵 1개 (작은 버전)
✅ 2v2 (또는 5v5)
✅ 15분 게임 한 판 완결
✅ Possession + Bond + Asymmetric Kit + Servant Death 4시스템
✅ 승패 판정
❌ 상점, 랭크, 채팅 없음
❌ 결제 없음
❌ 보스 PvE는 Outer Ring 미니보스만
```

# Phase 6 — 다음 결정점

여기까지가 디자인 좌표계와 핵심 무브의 1차 윤곽이다.
다음으로 정해야 하는 것들 — 답에 따라 모든 후속 시스템이 결정된다:

## 1. Servant의 정체성

너는 어떻게 보고 있어?

- (a) **펫 같은 별개 생명체**
- (b) **분신·그림자**
- (c) **소환된 영혼·도구**
- (d) **AI 동반자**

→ 이게 세계관, 비주얼, 죽음 페널티 톤을 다 결정한다.

## 2. 데모 페어 2개의 선택

- **Pair A** (즉발 × 설치형) — 격투 심리전 결
- **Pair B** (이속 × 탱커) — 미끼/안전망 결

→ 두 개 다 만들 시간은 없다. 하나에 올인하고 그 안에서 두 페어를 변주해야 한다.

## 3. PvE 강도 결정

- (a) **5분 짧은 미니보스** (라이트 PvE)
- (b) **게임 한 판에 1마리 큰 보스 (15분급)** — 엘든링 결
- (c) **PvE 자체를 빼고 모두 PvP**

→ 이게 10분 한 판 vs 30분 한 판을 가르는 분기.

## 4. 빙의 전환 비용

- (a) **0.5초 글로벌 쿨** (현재 안)
- (b) **더 비싸게** — 스킬 캐스팅 중엔 못 바꿈, 피격 시 락

→ 결정의 무게를 어디까지 키울지.

새 시스템을 추가할 때마다 이 체크리스트를 통과시킨다:

```

[ ] 어느 층위를 약화시키는가? (트레이드오프 명시)
[ ] 데모 v0.1 정의에 들어가는가, 아니면 후순위인가?
[ ] Asymmetric Kit 원칙을 깨지 않는가?
[ ] Bond 시스템과 충돌하지 않는가?
[ ] Possession 전환의 무게를 흐리지 않는가?
```
