다섯 명이 4주 동안 자체 DX9 엔진으로 마인크래프트 던전스를 모작했습니다. 저는 팀장으로 git 세팅부터 PR 135건 머지, 리소스 일정과 팀원 간 조율까지 협업의 뼈대를 잡았고, 개발로는 전용 서버(초당 20회 상태 동기화)와 엔더드래곤 보스전을 맡아 같은 와이파이에서 팀원 4명과 실제 레이드를 시연했습니다. 중도에 UI 담당의 작업이 레거시로 남았을 때는 팀장으로서 인수해 직접 재구현했습니다 — 팀의 구멍을 메우는 것도 팀장의 일이라는 걸 배운 구간입니다.

### 구현

- **[구조] Base/Engine/Client/Server 분리** — `CBase` AddRef/Release 수동 reference count(COM 스타일)로 엔진 객체 생명주기 관리, GameObject/Component + `CProtoMgr`의 Prototype/Clone으로 반복 생성 비용 절감 (Unity Prefab·UE CDO에 대응하는 개념을 몸으로 먼저 배움)
- **[담당] TCP 소켓 서버(select) + 20 tick 스냅샷 동기화** — Shared `PacketDef`로 login/input/StateSnapshot/projectile/damage/DragonSync 등 패킷 계약 명시(구조체 크기 static_assert 검증), 틱마다 상태 브로드캐스트. 이때 select 모델의 한계를 체감한 것이 Winters에서 IOCP(워커 4+틱 1)로 넘어간 이유
- **[담당] 엔더드래곤 보스전** — 블럭 vertex 로컬 좌표 조작으로 날개 생성, IK solver + sin파 연결 움직임으로 물리 시뮬레이션풍 드래곤 구현, 서버 권위 브로드캐스트 동기화
- **[담당] 블럭 배치 렌더링 — 측정으로 선택한 최적화**: 배치 버퍼로 드로우콜을 묶는 방식과 쿼드트리 프러스텀 컬링을 **둘 다 구현해 비교**했고, 이 씬에서는 배치가 더 좋은 수치가 나와 배치를 채택하고 컬링은 제외했습니다 — "정석"이 아니라 측정이 결정
- 그 외: ImGui 맵 에디터(멀티 스테이지·트리거 스폰), 파티클, HLSL 포스트프로세스 6종, 파도 스테이지(Sin파/Gerstner 구현 — FFT는 시도 후 범위 조정을 인정)

### ▶ 더 깊이 — 트레이드오프와 뒷이야기 (토글)

- **협업의 실제**: 다섯 명이 한 레포에서 4주 — 충돌은 기술이 아니라 규약으로 줄었다. 브랜치 전략과 PR 리뷰 강제, 리소스 일정과 코드 일정의 분리, 그리고 팀원 간 갈등을 팀장이 조율하는 것이 마감을 지킨 이유였다.
- **설계 일화 — 버그의 디자인 승격**: 드래곤 돌진이 의도(1회+꼬리치기)와 달리 2연속 돌진으로 동작했는데, 플레이해 보니 더 위협적이라 채택했다. 구현 중 발견을 기획으로 승격한 경험.
- **미해결 버그의 사후 규명 — 드래곤 동기화 버그**: 플레이 시간이 지날수록 드래곤의 공격이 **여러 마리가 공격하는 것처럼 중첩되어 보이는** 동기화 버그가 있었고, 당시에는 원인을 닫지 못한 채 마감했습니다. 프로젝트가 끝난 뒤 실코드를 다시 열어 원인을 규명했습니다 — ①공격 연출의 권위가 서버와 클라이언트 양쪽에 있어 같은 공격이 두 경로로 발화하는 **이중 권위 에코** ②클라이언트 로컬 타이머가 서버 틱과 어긋나며 지난 패턴의 연출이 누적되는 **타이머 드리프트** ③스테이지 트리거의 재장전. Winters는 이 사고의 구조적 재발 방지로 설계됐습니다: **모든 전투 사실은 서버 이벤트/스냅샷으로만 생성**되고 클라이언트는 연출만 담당하며, 시퀀스 규율과 이벤트 적용 단일 지점(EventApplier)이 이중 발화를 원천 차단하고, 300틱 결정성 해시가 회귀를 감시합니다. "버그를 고쳤다"가 아니라 **"버그가 불가능한 구조로 다시 지었다"**는 것이 이 서사의 결론입니다.
- **수동 ref-count의 교훈**: Release 누락 한 번이 어디서 터질지 모르는 누수가 된다는 걸 팀 코드에서 겪었다 — Winters에서 RAII+세대 핸들로 같은 문제를 구조적으로 재해결한 출발점.
- **이 프로젝트가 Winters로 이어진 것**: 패킷 계약·스냅샷 동기화 → GameCommand 검증·FlatBuffers Snapshot/Event. 드래곤 이중 권위 사고 → 서버 단일 진실. 그리고 "프레임워크가 정해준 구조 안에서는 틱 제어·복제 필터를 설계할 수 없다"는 한계 인식이 자체 엔진의 출발점.
