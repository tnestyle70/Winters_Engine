## 0. 이 분야를 박사로 본다는 것

게임 AI 분야에서 "구현"과 "기여"의 경계는 가이드 §1의 어떤 분야보다 흐려지기 쉽다. 봇이 라인을 밀고 미니언을 잡고 챔피언과 교전하는 데모는 강렬하지만, 그것은 **엔지니어링 결과물**이지 박사 기여가 아니다. "나는 DQN/PPO/MCTS 봇을 구현했다"는 명제(thesis)가 아니다. 박사 기여는 항상 다음 형태를 띤다: *"[새 기법 X]를 사용하면 [게임 AI 특유의 제약 C — 예: 결정론적 추론, ms 단위 예산, 디자이너 제어 가능성] 하에서 [목표 Y]를 [기존 SOTA 대비 정량적 개선 Z]로 달성한다."* OpenAI Five와 AlphaStar가 학술적으로 인용되는 이유는 "Dota2/StarCraft를 풀었다"가 아니라, **거대 부분관측·장기 신용할당·실시간 행동공간이라는 풀리지 않던 제약 하에서 self-play 기반 학습이 작동함을 정량적으로 보였기 때문**이다. Winters의 testbed 가치도 동일하다: "결정론 서버라는 강한 제약이 학습 기반 AI에 부과하는 비용을 정량화하고, 그 비용을 줄이는 새 기법을 제안"하는 순간 구현이 기여로 바뀐다.

게임 AI 박사는 가이드 §5의 기여 유형 중 주로 **1(새 알고리즘)·3(새 시스템/아키텍처)·5(새 경험적 발견)** 의 조합이며, **6(새 벤치마크/평가 환경)** 이 의외로 강력하다(ALE, OpenAI Gym, SMAC, MineRL이 모두 이 유형으로 분야를 움직였다).

## 1. 심층 강화학습 (Deep Reinforcement Learning)

게임 AI 박사의 가장 뜨겁고 가장 경쟁이 심한 영역. 핵심 긴장점은 명확하다: **deep RL은 샘플을 수억~수십억 step 먹는데, 게임 AI는 결정론·실시간·제어가능성이라는 산업 제약을 동시에 요구한다.** 이 간극이 곧 open problem의 원천이다.

### 1.1 핵심 원리 (이론/수학)

**Markov Decision Process (MDP).** 강화학습의 수학적 기반은 MDP 튜플 `(S, A, P, R, γ)`이다.
- `S`: 상태 공간, `A`: 행동 공간
- `P(s' | s, a)`: 전이 확률 (Winters에서는 결정론 GameSim이 곧 `P`이며, 이것이 deterministic — 아래 1.4의 핵심)
- `R(s, a)`: 보상 함수
- `γ ∈ [0,1)`: 할인율 (discount factor)

목표는 누적 할인 보상의 기댓값을 최대화하는 정책 `π(a | s)`를 찾는 것이다:

```text
J(π) = E_π [ Σ_{t=0}^{∞} γ^t · R(s_t, a_t) ]
```

**Value function과 Bellman 방정식.** 정책 `π` 하의 상태가치와 행동가치:

```text
V^π(s) = E_π [ Σ_k γ^k r_{t+k} | s_t = s ]
Q^π(s,a) = E_π [ Σ_k γ^k r_{t+k} | s_t = s, a_t = a ]
```

Bellman 기대 방정식 (정책 평가):

```text
V^π(s) = Σ_a π(a|s) Σ_{s'} P(s'|s,a) [ R(s,a) + γ V^π(s') ]
```

Bellman 최적 방정식 (최적 가치):

```text
Q*(s,a) = Σ_{s'} P(s'|s,a) [ R(s,a) + γ max_{a'} Q*(s',a') ]
```

이 방정식의 우변은 contraction mapping(수축 사상)이며, 고정점이 유일하게 존재한다 — 이것이 value iteration의 수렴을 보장하는 이론적 핵심이다(γ < 1일 때 sup-norm에서 γ-contraction).

**Q-learning (off-policy TD control).** 모델(`P`)을 모를 때 sample로 Bellman 최적 방정식을 부트스트랩하는 업데이트:

```text
Q(s,a) ← Q(s,a) + α [ r + γ max_{a'} Q(s',a') − Q(s,a) ]
                       └──────── TD target ────────┘
```

대괄호 안이 TD error(시간차 오차) `δ`. tabular 환경에서 모든 (s,a)를 무한히 방문하고 학습률 조건(Robbins-Monro)을 만족하면 `Q → Q*`로 수렴함이 증명되어 있다.

**Policy Gradient (REINFORCE).** 가치를 거쳐 정책을 유도하는 대신, 정책 `π_θ`를 직접 파라미터 `θ`로 두고 `J(θ)`를 경사상승. Policy Gradient Theorem:

```text
∇_θ J(θ) = E_{π_θ} [ ∇_θ log π_θ(a|s) · Q^{π_θ}(s,a) ]
```

REINFORCE는 `Q`를 Monte Carlo return `G_t`로 추정:

```text
∇_θ J(θ) ≈ Σ_t ∇_θ log π_θ(a_t|s_t) · G_t
```

이 추정량은 unbiased지만 **분산이 매우 크다**. baseline `b(s)`를 빼도 unbiased가 유지되므로(`E[∇log π · b(s)] = 0`), 분산 감소를 위해 `b(s) = V(s)`를 쓴다 → advantage `A(s,a) = Q(s,a) − V(s)`.

**Actor-Critic.** Actor `π_θ`(정책)와 Critic `V_w`(가치 추정)를 동시 학습. Critic이 advantage를 추정해 actor의 분산을 줄인다:

```text
δ = r + γ V_w(s') − V_w(s)          # TD error = advantage 추정
θ ← θ + α_θ · ∇_θ log π_θ(a|s) · δ   # actor 업데이트
w ← w + α_w · δ · ∇_w V_w(s)         # critic 업데이트
```

GAE (Generalized Advantage Estimation)는 `δ`들을 `λ`로 지수가중 합산해 bias-variance를 조절한다:

```text
A_t^{GAE(γ,λ)} = Σ_{l=0}^{∞} (γλ)^l · δ_{t+l}
```

### 1.2 대표 기존 연구 (SOTA·핵심 논문)

- **Sutton & Barto, *Reinforcement Learning: An Introduction* (2nd ed., 2018).** 분야의 정전(canonical text). MDP·Bellman·TD·policy gradient의 모든 기초가 여기 있다. 박사 서론의 §배경은 이 책을 인용해 독자를 끌고 간다.
- **Mnih et al., "Human-level control through deep reinforcement learning," *Nature* (2015) — DQN.** Deep RL의 출발점. 두 가지 안정화 트릭이 핵심 기여다: (1) **experience replay** — 상관된 연속 샘플을 버퍼에 저장 후 무작위 미니배치로 학습해 i.i.d. 가정을 근사, (2) **target network** — TD target의 `max` 항을 느리게 갱신되는 별도 네트워크 `Q_{θ⁻}`로 계산해 부트스트랩의 발산을 억제. 손실:

  ```text
  L(θ) = E_{(s,a,r,s')~D} [ ( r + γ max_{a'} Q_{θ⁻}(s',a') − Q_θ(s,a) )^2 ]
  ```

- **Schulman et al., "Proximal Policy Optimization Algorithms" (2017) — PPO.** 현재 게임/로보틱스 deep RL의 사실상 표준. TRPO의 KL 제약을 clipped surrogate로 근사해 구현 난이도를 크게 낮췄다. 확률비 `r_t(θ) = π_θ(a_t|s_t) / π_{θ_old}(a_t|s_t)`에 대해:

  ```text
  L^{CLIP}(θ) = E_t [ min( r_t(θ)·A_t,  clip(r_t(θ), 1−ε, 1+ε)·A_t ) ]
  ```

  clip이 정책을 한 번에 너무 크게 바꾸지 못하게 막아(trust region 근사) 안정성을 준다. OpenAI Five의 백본이 PPO다.
- **Silver et al., "Mastering the game of Go..." *Nature* (2016) — AlphaGo.** policy/value network + MCTS + 인간 기보 supervised + self-play RL의 결합. 게임 AI에서 "탐색(search)과 학습(learning)의 결합"이라는 패러다임을 정립.
- **Silver et al., "Mastering the game of Go without human knowledge," *Nature* (2017) — AlphaZero.** 인간 데이터 0. 순수 self-play. MCTS를 **정책 개선 연산자(policy improvement operator)** 로 보고, 네트워크가 MCTS 방문분포를 모방하도록 학습하는 단일 루프로 통합(§2.1과 연결). 단일 알고리즘으로 Go·Chess·Shogi 정복.
- **Schrittwieser et al., "Mastering Atari, Go, Chess and Shogi by planning with a learned model," *Nature* (2020) — MuZero.** 환경 모델 `P`를 **모르는 채로** 가치·정책·보상을 예측하는 latent dynamics model을 학습해 MCTS를 돌린다. "규칙을 모르는 환경에서의 model-based planning"을 연 기여. (Winters 연결: 결정론 GameSim이라는 *완벽히 알려진 모델*이 있으므로, MuZero의 학습된 모델 대신 실제 GameSim을 planning 모델로 쓰는 변형이 자연스러운 연구 각도가 된다 — 1.4 참조.)
- **Berner et al., "Dota 2 with Large Scale Deep Reinforcement Learning" (2019) — OpenAI Five.** MOBA 도메인의 정점. 부분관측·장기 horizon(수만 step)·거대 행동공간을 LSTM 정책 + 대규모 분산 PPO + self-play로 해결. **surgery**(아키텍처를 바꾸면서도 학습을 이어가는 기법)와 **team spirit**(개인 보상 vs 팀 보상 가중치 `τ`를 스케줄링) 같은 실무 기여 포함. Winters MOBA 봇 연구의 가장 직접적인 선행 연구.
- **Vinyals et al., "Grandmaster level in StarCraft II using multi-agent reinforcement learning," *Nature* (2019) — AlphaStar.** 거대 구조화 행동공간 + 부분관측 + 비대칭 종족. **League training**(다양한 상대 풀을 유지해 nash 근사·전략 cycle 붕괴 방지)이 핵심 기여. self-play의 비추이성(non-transitivity, 가위바위보식 전략 순환) 문제를 정면으로 다룬다.

### 1.3 자료구조/알고리즘 (의사코드)

PPO actor-critic 학습 루프 (게임 self-play 맥락의 골격):

```text
initialize policy π_θ, value V_w
for iteration = 1, 2, ... do
    # 1) Rollout: 현재 정책으로 N step 경험 수집 (병렬 환경 K개)
    D ← []
    for each parallel env do
        for t = 1..T do
            a_t ~ π_θ(· | s_t)
            s_{t+1}, r_t ← env.step(a_t)      # Winters: deterministic GameSim tick
            store (s_t, a_t, r_t, log π_θ(a_t|s_t)) in D
    # 2) Advantage 계산 (GAE)
    for t = T..1 do
        δ_t   = r_t + γ·V_w(s_{t+1}) − V_w(s_t)
        A_t   = δ_t + γλ·A_{t+1}
        Ret_t = A_t + V_w(s_t)               # value target
    # 3) 여러 epoch 동안 미니배치 SGD
    θ_old ← θ
    for epoch = 1..E do
        for minibatch B ⊂ D do
            r_ratio = exp(log π_θ(a|s) − log π_{θ_old}(a|s))
            L_clip  = mean( min(r_ratio·A, clip(r_ratio,1−ε,1+ε)·A) )
            L_v     = mean( (V_w(s) − Ret)^2 )
            L_ent   = mean( entropy(π_θ(·|s)) )         # 탐험 유도
            loss    = −L_clip + c1·L_v − c2·L_ent
            θ, w ← Adam_step(∇ loss)
```

자료구조 노트:
- **Replay buffer** (DQN 계열): 보통 ring buffer. prioritized replay는 TD error 크기에 비례한 sampling을 위해 **sum-tree**(segment tree) 사용 → `O(log N)` 샘플/업데이트.
- **분산 아키텍처**: actor(rollout 생성) / learner(gradient) 분리. OpenAI Five는 수천 GPU·CPU. 핵심 자료구조는 actor→learner 경험 큐와 파라미터 브로드캐스트.

### 1.4 박사급 novel 기여 각도 (open problems)

게임 AI, 특히 **결정론 server-authoritative MOBA**라는 Winters 제약이 만드는 진짜 open problem들:

1. **결정론적 추론(deterministic inference)이라는 제약.** server-authoritative 결정론 시뮬레이션은 모든 클라이언트가 비트 단위로 같은 결과를 봐야 한다. 그런데 학습된 NN 정책 추론은 (a) 부동소수점 비결정성(SIMD reduction 순서, FMA, 플랫폼별 `libm`), (b) GPU 비결정성을 갖는다. **Open problem:** "결정론을 깨지 않으면서 학습된 정책을 서버 tick 안에서 추론하는 방법" — fixed-point 양자화 추론, 결정론적 정수 행렬곱, 또는 정책을 결정론적 lookup/decision-list로 distillation. 이것은 가이드 §5의 유형 3(시스템) + 4(증명: "비트 재현성 보장")의 강한 결합이며 거의 다뤄지지 않은 틈이다.

2. **실시간 추론 예산(ms budget).** 서버 tick이 예: 33ms/30Hz라면 수십~수백 봇의 추론이 그 안에 끝나야 한다. **Open problem:** "추론 예산 B(ms) 하에서 품질-지연 Pareto front를 최적화하는 정책 압축/조기종료(anytime inference)". metric이 명확(ms, 봇 수, 승률)해서 박사 평가에 강하다.

3. **Sample efficiency.** OpenAI Five는 수백 년치 게임을 학습했다. 인디/중소 스튜디오가 재현 불가능한 규모다. **Open problem:** "완벽히 알려진 결정론 모델(GameSim 자체)을 planning에 직접 활용해 model-free RL의 샘플 수를 수십 배 줄이기" — MuZero가 모델을 *학습*했다면, Winters는 모델이 *공짜*다. 이 비대칭이 novel 기여 각도다(model-based RL + 실제 시뮬레이터 = AlphaZero식 search를 MOBA에 이식하되 학습 모델 없이).

4. **디자이너 제어 가능성(controllability).** 산업 게임 AI의 진짜 요구는 "최강 봇"이 아니라 "Bronze~Diamond 난이도를 *연속적으로* 조절 가능하고, *특정 스타일*(공격적/수비적)을 강제 가능한 봇"이다. **Open problem:** "단일 학습된 정책에서 난이도·스타일을 연속 파라미터로 조건화(conditional policy)하고, 그 파라미터가 실제 행동 지표(KDA, 교전 빈도)와 단조 관계를 갖도록 보장". 학술적으로 과소연구된, 그러나 IEEE CoG/ToG가 좋아하는 기여.

5. **Sim-to-game 전이.** 학습 시점의 밸런스 패치 v1.0과 배포 시점의 v1.3이 다르다(아이템·챔피언 수치 변경). **Open problem:** "밸런스 패치에 robust한 정책, 또는 패치 delta만으로 빠르게 fine-tune하는 방법". domain randomization을 게임 밸런스 공간에 적용.

> 주의(가이드 §11 약한 baseline 금지): "내 RL 봇이 scripted 봇을 이겼다"는 약한 baseline이다. SOTA baseline은 PPO self-play, 그리고 가능하면 OpenAI Five식 league여야 한다.

### 1.5 Thesis statement 예시

- "결정론 정수 추론으로 distillation된 정책을 사용하면, server-authoritative MOBA에서 비트 단위 재현성을 깨지 않고 PPO self-play 봇을 cross-platform으로 배포할 수 있으며, float 추론 대비 승률 손실 2% 이내·tick당 추론 비용 5배 감소를 달성한다."
- "완벽히 알려진 결정론 GameSim을 planning 모델로 직접 사용하는 AlphaZero식 search-augmented 정책은, model-free PPO 대비 동일 승률 도달에 필요한 시뮬레이션 step을 한 자릿수 배 줄인다(MOBA 1v1 라인전 도메인)."
- "난이도 스칼라 `d ∈ [0,1]`로 조건화된 단일 정책은, `d`에 대해 단조 증가하는 승률과 사람 플레이어가 지각하는 난이도(user study)를 동시에 만족시키며, 난이도별 개별 정책 학습 대비 학습 비용을 N배 절감한다."

### 1.6 평가 방법 (지표)

가이드 §7을 게임 RL에 특화:
- **성능/품질**: 승률(vs 고정 상대 풀, vs 사람), TrueSkill/Elo, 도메인 지표(KDA, GPM, objective control rate).
- **샘플 효율**: 목표 승률 도달까지의 environment step 수 (학습 곡선, x축 log scale).
- **실시간성**: tick당 추론 시간(ms) 분포(평균·p99), 동시 봇 수 대비 곡선(가이드 §7 scalability).
- **결정론**: cross-platform/cross-run 비트 재현률(%), divergence까지의 평균 tick 수.
- **Ablation (필수)**: replay/target net/GAE/league 각 구성요소 제거 시 성능 하락. "무엇이 효과의 원천인가"를 증명.
- **통계**: 최소 5 seed, 신뢰구간/IQM(interquartile mean) 보고(평균만 ❌). non-transitivity 때문에 단일 상대 승률은 신뢰 못 함 → 상대 풀 전체에 대한 행렬.
- **재현성·Threats to validity**: 하드웨어·하이퍼파라미터·밸런스 버전 명시. "학습한 밸런스에 overfit"이 가장 큰 타당성 위협.

### 1.7 Winters 엔진 연결점 (testbed)

Winters는 deep RL 연구에 **이상적이면서 동시에 까다로운** testbed다. 그 까다로움이 곧 연구 기여의 원천이다.

- **결정론 GameSim = 완벽한 환경 모델.** `Shared/GameSim/Systems/`의 시뮬레이션은 deterministic `step()` 함수 그 자체다(MDP의 `P`). RL 환경 래퍼를 GameSim 위에 직접 올릴 수 있고(headless·고속), 동시에 1.4-(3)의 "model이 공짜"인 search 연구를 가능케 한다.
- **AI = 입력 생산자라는 구조적 제약이 그대로 RL action space가 된다.** handoff 문서가 못 박은 대로 봇은 `Move / BasicAttack / CastSkill / LevelSkill / ...` 의 `GameCommand`만 emit한다(`Shared/GameSim/Systems/ICommandExecutor.h`). 즉 **action space가 이미 깔끔하게 정의·검증**되어 있다(사람과 동일 경로). RL 정책의 출력 = 이 command 분포. `CDefaultCommandExecutor`가 reject하면 그것이 곧 invalid action masking 신호.
- **self-play 인프라가 이미 존재.** `Server/Private/Game/GameRoom.cpp`의 `Phase_ServerBotAI → Phase_ExecuteCommands` 루프(handoff anchor: `GameRoom.cpp:805` `Phase_ServerBotAI`, `:810` `CBotLaneAISystem::Execute`)가 봇 command를 사람 command와 같은 큐(`m_pendingExecCommands`)로 흘린다. 여러 봇을 양 팀에 배치하면 self-play 대전 환경이 즉시 성립.
- **측정 인프라 = AIDebugPanel.** `Client/Private/UI/AIDebugPanel.cpp`(state/target/score/cooldown 출력)와 `SnapshotBuilder`의 `BotLaneAIComponent` 인코딩(handoff anchor: `SnapshotBuilder.cpp:229`)이 곧 RL 보상·관측의 디버그 창. 가이드 §7 "측정 인프라가 평가의 절반"이 그대로 적용.
- **연구 긴장점(핵심).** 결정론 서버이므로 (a) NN 추론을 tick 안에 넣으면 비결정성 위험, (b) 추론 예산이 실시간으로 묶임. 이 긴장이 1.4-(1)(2)의 thesis를 *Winters에서만 선명하게* 검증 가능하게 만든다. 기존 OpenAI Five/AlphaStar는 결정론 재현 제약이 없었으므로, "결정론 제약 하 deep RL"은 Winters가 제공하는 차별적 기여 공간이다.

## 2. Monte Carlo Tree Search (MCTS)

탐색(search) 기반 의사결정의 대표. RL이 "경험으로 정책을 학습"한다면 MCTS는 "현재 상태에서 모델을 굴려 즉석 계획"한다. AlphaZero 이후 둘은 한 몸이 되었다. MOBA처럼 **거대·연속·동시(simultaneous) 행동공간**에서는 고전 MCTS가 그대로는 안 통한다 — 그것이 open problem.

### 2.1 핵심 원리 (이론/수학)

MCTS는 행동가치를 시뮬레이션 통계로 추정하며 탐색 트리를 점진적으로(anytime) 키운다. 한 iteration은 4단계:

```text
1. Selection : 루트에서 트리 정책(예: UCT)으로 leaf까지 내려감
2. Expansion : leaf에서 자식 노드 하나(또는 전부) 추가
3. Simulation: 추가된 노드에서 종단까지 rollout(기본은 random playout)
4. Backprop  : rollout 결과를 경로상 모든 노드의 (N, W) 통계에 역전파
```

**UCT (Upper Confidence bounds applied to Trees).** selection은 multi-armed bandit으로 본다. 각 자식 행동 `a`에 대해 UCB1 공식으로 점수를 매겨 argmax 선택:

```text
UCB1(s,a) = Q(s,a) + c · sqrt( ln N(s) / N(s,a) )
            └ exploitation ┘   └──── exploration ────┘
```

- `Q(s,a) = W(s,a)/N(s,a)`: 평균 보상(착취)
- `N(s)`: 부모 방문 수, `N(s,a)`: 그 자식 방문 수
- `c`: 탐험 상수(보통 `√2` 부근). 적게 방문된 행동에 보너스를 줘 탐험-착취 균형.

UCB1은 cumulative regret이 `O(ln T)`로 최적임이 증명되어 있고(Auer et al. 2002), UCT는 이를 트리에 재귀 적용해 **충분한 시뮬레이션에서 최적 행동으로 수렴**함이 보장된다(Kocsis & Szepesvári 2006).

**PUCT (Predictor + UCT, AlphaZero).** rollout 대신 **사전확률 `P(s,a)`(정책망)**로 탐색을 가이드하고, simulation의 종단값을 **가치망 `V(s)`**로 대체:

```text
PUCT(s,a) = Q(s,a) + c_puct · P(s,a) · sqrt( Σ_b N(s,b) ) / (1 + N(s,a))
```

핵심 통찰: AlphaZero에서 **MCTS는 정책 개선 연산자**다. 한 상태에서 충분히 search한 뒤의 *방문분포* `π_MCTS(a) ∝ N(s,a)^{1/τ}`는 raw 정책망 `P`보다 강하다. 따라서 학습은 `P → π_MCTS`, `V → 게임 결과 z`로 향한다. 이것이 §1.2 AlphaZero/MuZero를 RL과 MCTS의 *교집합*에 놓는 이유다.

### 2.2 대표 기존 연구 (SOTA·핵심 논문)

- **Coulom, "Efficient Selectivity and Backup Operators in Monte-Carlo Tree Search" (2006).** MCTS의 출발점. Monte-Carlo 평가와 트리 탐색의 결합을 정식화(Go 프로그램 Crazy Stone).
- **Kocsis & Szepesvári, "Bandit based Monte-Carlo Planning" (2006) — UCT.** UCB1을 트리에 적용하고 수렴성을 분석한 정전. 이후 거의 모든 MCTS가 UCT 변형.
- **Browne et al., "A Survey of Monte Carlo Tree Search Methods," *IEEE ToG/TCIAIG* (2012).** MCTS 박사 연구의 필수 인용. 수십 종 변형(RAVE, AMAF, progressive widening 등)과 enhancement를 분류. 박사 §관련연구의 뼈대.
- **Silver et al. (2016, 2017) — AlphaGo/AlphaZero.** PUCT + 신경망 가이드(§1.2). MCTS를 학습과 결합한 패러다임.
- **Schrittwieser et al. (2020) — MuZero.** 학습된 모델로 MCTS(§1.2). simulation에 실제 환경이 필요 없게 만듦.
- **연속/동시 행동공간 확장**: progressive widening / double progressive widening(Couëtoux et al. 2011)이 연속 행동에서 자식 수를 `N^α`로 제한해 트리 폭발을 막는 표준 기법. 동시수(simultaneous-move) 게임에는 decoupled-UCT, regret-matching 기반(SM-MCTS) 변형. MOBA는 이 두 어려움(연속 + 동시)을 동시에 가짐 → 2.4 open problem.

### 2.3 자료구조/알고리즘 (의사코드)

```text
function MCTS(root_state, budget):
    root ← Node(root_state)
    while within_budget(budget):           # anytime: 시간/시뮬레이션 수 예산
        node ← root
        # 1) Selection
        while node.is_fully_expanded() and not node.is_terminal():
            node ← argmax_{child} UCB1(node, child)     # or PUCT
        # 2) Expansion
        if not node.is_terminal():
            node ← node.add_one_untried_child()
        # 3) Simulation (default policy = random, or value network)
        reward ← rollout(node.state)         # AlphaZero: reward ← V_network(node.state)
        # 4) Backpropagation
        while node ≠ null:
            node.N += 1
            node.W += reward
            reward ← flip_if_adversarial(reward)   # 2인 zero-sum이면 부호 반전
            node ← node.parent
    return argmax_{child of root} child.N    # 가장 많이 방문된 행동(robust child)

structure Node:
    state
    parent, children[]
    untried_actions[]
    N (visit count), W (total value)
    P (prior, PUCT일 때)        # 정책망 사전확률
```

자료구조 노트:
- 트리 노드는 명시적 포인터 트리. 메모리 압박 시 transposition table(상태 해시 → 노드)로 상태 합치기.
- **anytime 보장**: 예산이 시간(ms)이든 시뮬레이션 수든, 루프를 언제 끊어도 현재까지의 최선(`robust child`)을 반환 — 실시간 게임에 결정적 장점.
- 병렬화: root parallelization(독립 트리 N개 후 통계 합산), leaf/tree parallelization(virtual loss로 같은 경로 중복 방지).

### 2.4 박사급 novel 기여 각도 (open problems)

1. **거대·연속·동시 행동공간(MOBA의 본질).** MOBA의 한 tick 행동은 (이동 방향 = 연속) × (스킬 슬롯 = 이산) × (조준 위치 = 연속) 의 곱이며, 양 팀이 *동시에* 행동한다. 고전 UCT는 이산·순차(turn-based) 가정. **Open problem:** "MOBA tick 행동공간에 대한 double progressive widening + simultaneous-move regret matching의 결합, 그리고 그 수렴 분석". 가이드 §5 유형 1+4.
2. **실시간/anytime 예산.** 33ms tick 안에 의미 있는 트리를 만들 수 있는가? **Open problem:** "고정 ms 예산 하에서 search 깊이/폭을 적응적으로 배분(meta-reasoning)해 품질을 최대화". metric: 예산 대비 행동 품질 곡선.
3. **신경망 가이드 비용 vs 효과.** PUCT는 매 노드 정책/가치망 추론이 필요 — 게임 tick 예산에서 NN 호출이 병목. **Open problem:** "추론 호출 예산이 제한될 때, 어느 노드에서만 NN을 부를지(selective evaluation) 결정하는 정책". 결정론 추론 제약(§1.4-1)과 합쳐지면 Winters 특화 기여.
4. **결정론 search.** MCTS rollout이 결정론 GameSim을 쓰면 재현 가능하지만, random playout의 RNG·노드 확장 순서가 비결정성을 유발할 수 있다. **Open problem:** "비트 재현 가능한 server-side MCTS"(seed 관리 + tie-break 규칙). §1.4-1과 쌍을 이룸.

### 2.5 Thesis statement 예시

- "Double progressive widening과 simultaneous-move regret matching을 결합한 MCTS 변형은, MOBA의 연속·동시 행동공간에서 고정 ms 예산 하에 decoupled-UCT 대비 교전 승률을 유의하게 개선한다."
- "노드별 NN 평가를 선택적으로만 수행하는 budget-aware PUCT는, 모든 노드를 평가하는 표준 PUCT와 동일 행동 품질을 NN 추론 호출 수 K배 절감으로 달성한다(서버 tick 예산 33ms)."

### 2.6 평가 방법 (지표)

- **행동 품질**: 고정 상대/사람 대비 승률, 미니게임(라인전·한타) 시나리오에서의 결과 지표.
- **예산 대비 곡선(핵심)**: x=시뮬레이션 수 또는 ms, y=품질. anytime 특성을 보이는 그래프.
- **NN 호출 효율**: PUCT 변형이면 추론 호출 수 대비 품질.
- **Ablation**: progressive widening 계수 `α`, `c_puct`, virtual loss, transposition table 유무.
- **결정론**: rollout 재현률.
- **Baseline**: vanilla UCT, decoupled-UCT, full PUCT. (scripted 봇은 baseline으로 약함.)
- **Threats to validity**: simulation 모델(GameSim) 충실도, 상대 풀의 다양성.

### 2.7 Winters 엔진 연결점 (testbed)

- **결정론 GameSim = 공짜 정확 모델 = MCTS의 꿈.** MCTS의 약점은 "정확한 forward model 필요". Winters는 그 모델이 곧 게임 엔진이다(`Shared/GameSim/`). MuZero처럼 모델을 학습할 필요 없이 실제 `step()`을 rollout에 사용 → search 정확도가 곧 게임 정확도.
- **action space가 이미 `GameCommand`로 이산화/검증됨.** MCTS의 자식 = 가능한 command 집합. `CDefaultCommandExecutor`의 reject가 곧 불법 자식 가지치기. (단, 연속 조준/이동을 어떻게 자식으로 양자화할지가 2.4-1의 연구거리.)
- **anytime이 서버 tick 예산과 정합.** `GameRoom.cpp`의 `Phase_ServerBotAI`가 tick당 한 번 호출 — 그 예산(ms) 안에서 MCTS를 anytime으로 돌리고 `robust child` command를 emit하면, 기존 `CBotLaneAISystem`의 룰 기반 결정을 search 기반으로 교체하는 깔끔한 슬롯이 된다(같은 command 인터페이스 유지).
- **AlphaZero식 통합 경로.** §1(PPO/self-play) + §2(PUCT search)를 Winters에서 한 시스템으로 묶으면, MOBA 도메인에서의 "search-augmented self-play"라는 통합 학위논문의 핵심 testbed가 된다(종합 절 참조).

## 3. 진화·유전 알고리즘 (Genetic / Evolutionary Computation)

gradient 없이 모집단(population)을 진화시키는 black-box 최적화·탐색 패러다임. 게임 AI에서 강점은 (a) **미분 불가능/비결정적 목적함수**(승률, 재미, 다양성)에 직접 적용, (b) **다양성 있는 해 집합**을 한 번에 생성(Quality-Diversity), (c) **절차적 콘텐츠 생성(PCG)·밸런싱**. RL이 한 개의 강한 정책을 찾는다면, 진화는 *여러 개의 흥미로운* 해를 찾는다.

### 3.1 핵심 원리 (이론/수학)

**Genetic Algorithm (GA).** 후보해를 유전체(genotype)로 인코딩하고, 적합도(fitness)에 따라 자연선택을 모사:

```text
1. 초기 모집단 P = { x_1, ..., x_μ } 무작위 생성
2. 각 개체 적합도 f(x_i) 평가
3. 선택(selection): 적합도 비례로 부모 선택 (룰렛휠, 토너먼트)
4. 교차(crossover): 두 부모 유전체를 섞어 자식 생성 (1-point, uniform 등)
5. 변이(mutation): 자식 유전체를 확률 p_m로 무작위 교란
6. 다음 세대로 교체, 2로 반복
```

**선택 압력(selection pressure)** 과 **다양성(diversity)** 의 균형이 핵심. 압력이 너무 세면 조기수렴(premature convergence), 약하면 진전 없음.

**Evolution Strategies (ES).** 실수 벡터 최적화에 특화. 핵심은 **search distribution `N(θ, σ²I)`** 자체를 진화. ES gradient 추정(Salimans et al. 2017, "Evolution Strategies as a Scalable Alternative to RL"):

```text
∇_θ E_{ε~N(0,I)} [ F(θ + σε) ] ≈ (1/(nσ)) Σ_{i=1}^{n} F(θ + σε_i) · ε_i
```

즉 파라미터를 가우시안 노이즈로 n번 교란해 평가하고, *적합도 가중 평균 방향*으로 `θ`를 이동. backprop이 없고 forward만 필요해 **대규모 병렬화에 극도로 친화적**(노이즈 seed만 통신). RL의 대안으로 제시됨.

**CMA-ES (Covariance Matrix Adaptation ES).** ES의 가장 강력한 변형. search 분포의 **공분산 행렬 `C`를 적응적으로 학습**해 탐색 타원체를 목적함수 지형에 맞춤(ill-conditioned·비분리 함수에 강함). 갱신 골격:

```text
샘플:   x_i = m + σ · N(0, C),  i=1..λ
선택:   상위 μ개로 평균 갱신  m ← Σ w_i x_{i:λ}
적응:   C ← (rank-μ + rank-one 업데이트),  σ ← (CSA: step-size 경로 길이로 조절)
```

저차원~중차원 연속 최적화의 사실상 표준 black-box optimizer.

**Quality-Diversity (QD) / MAP-Elites.** "가장 좋은 하나"가 아니라 **"행동 특성(behavior descriptor) 공간을 채우는, 각 칸에서 가장 좋은 해들의 아카이브"** 를 찾는 패러다임 전환. MAP-Elites(Mouret & Clune 2015, "Illuminating search spaces by mapping elites"):

```text
- behavior descriptor b(x) (예: NPC의 [공격성, 이동량])로 N차원 격자 정의
- 각 격자 칸은 그 행동 영역의 elite(최고 fitness) 1개 보관
- 무작위 elite를 변이 → b(x') 계산 → 해당 칸이 비었거나 더 나으면 교체
- 결과: "다양하면서 각자 우수한" 해의 지도(map)
```

QD는 게임 AI에서 결정적이다: "다양한 NPC 행동 레퍼토리", "다양한 레벨 집합"을 *명시적으로* 산출.

### 3.2 대표 기존 연구 (SOTA·핵심 논문)

- **Holland, *Adaptation in Natural and Artificial Systems* (1975).** GA의 기원·schema theorem.
- **Stanley & Miikkulainen, "Evolving Neural Networks through Augmenting Topologies" (2002) — NEAT.** 신경망의 *가중치와 구조(topology)를 함께* 진화. 세 핵심 기여: (1) **historical marking**(innovation number)으로 서로 다른 구조의 유전체를 정렬·교차, (2) **speciation**(종 분화)으로 새 구조를 보호, (3) **complexification**(최소 구조에서 점진적 성장). 게임 NPC 진화의 정전. 후속 HyperNEAT는 간접 인코딩으로 대규모 망을 진화.
- **Salimans et al., "Evolution Strategies as a Scalable Alternative to Reinforcement Learning" (2017).** ES가 Atari/MuJoCo에서 RL과 경쟁 가능하며 *훨씬 잘 병렬화*됨을 보임. backprop·value function 불필요, 희소·지연 보상에 강건.
- **Hansen, CMA-ES (Hansen & Ostermeier 2001 외).** 연속 black-box 최적화의 표준. 게임에서 AI 파라미터·콘텐츠 파라미터 튜닝에 직접 사용.
- **Mouret & Clune, "Illuminating search spaces by mapping elites" (2015) — MAP-Elites.** QD의 대표. 이후 **CMA-ME**(Fontaine et al. 2020, CMA-ES와 MAP-Elites 결합)로 발전.
- **PCG via Evolution / Search-Based PCG**: Togelius et al., "Search-Based Procedural Content Generation: A Taxonomy and Survey" (*IEEE ToG*, 2011). 적합도로 "플레이 가능성·난이도·다양성"을 인코딩해 레벨·맵·룰을 진화시키는 분야 정리. PCGML(머신러닝 기반)로 확장.

### 3.3 자료구조/알고리즘 (의사코드)

MAP-Elites (게임 NPC 다양성 생성에 직접 적용 가능한 형태):

```text
function MAP_ELITES(iterations):
    archive ← empty grid over behavior_space   # 칸 → (genotype, fitness)
    for i = 1..iterations:
        if i <= init_random:
            x ← random_genotype()
        else:
            parent ← random_elite_from(archive)
            x ← mutate(parent)                  # + 가능하면 crossover
        # 평가: 게임에서 NPC를 x로 실행
        perf, behavior ← simulate(x)            # Winters: GameSim self-play 결과
        cell ← discretize(behavior)             # 예: [공격성, 평균이동] → 격자 좌표
        if archive[cell] is empty or perf > archive[cell].fitness:
            archive[cell] ← (x, perf)
    return archive                              # 다양하고 각자 우수한 NPC 집합

structure Genotype:        # 예: 행동 파라미터 벡터 또는 NEAT 그래프
    params[] / network
```

GA 골격(밸런싱·튜닝용):

```text
P ← init_population(μ)
evaluate(P)                                     # 비싼 단계: self-play 매치
repeat:
    parents  ← select(P)                        # tournament(k)
    offspring ← crossover(parents)
    offspring ← mutate(offspring, p_m)
    evaluate(offspring)
    P ← survivor_select(P ∪ offspring, μ)       # elitism: 최고 개체 보존
until budget_exhausted
```

자료구조 노트:
- MAP-Elites archive: 저차원이면 dense N차원 배열, 고차원이면 해시맵 또는 CVT(centroidal Voronoi tessellation) 기반 칸.
- 평가가 병목(매 개체 = 여러 self-play 매치). 분산 평가 큐 + 캐싱이 핵심 엔지니어링.

### 3.4 박사급 novel 기여 각도 (open problems)

1. **다양성 있는 NPC 행동 레퍼토리.** 상용 게임은 "강한 봇 1개"가 아니라 "서로 다르게 플레이하는 봇 N개"가 필요(매판 같은 봇은 지루함). **Open problem:** "MOBA 챔피언별로, 행동 특성 공간(공격성/로밍/파밍 성향)을 채우는 QD 아카이브를 생성하고, 그 다양성이 사람이 지각하는 '봇 같지 않음'과 상관됨을 user study로 입증". 가이드 §5 유형 1+5.
2. **절차적 콘텐츠 생성(PCG).** 오픈월드 Winters의 지형/스폰/퀘스트 배치. **Open problem:** "플레이 가능성·난이도 곡선·다양성을 동시에 만족하는 콘텐츠를 QD로 생성하고, expressivity(생성 공간의 넓이)와 controllability(디자이너 의도 반영)를 정량화". search-based PCG + QD.
3. **자동 밸런싱.** 새 챔피언/아이템 수치를 넣으면 메타가 깨진다. **Open problem:** "self-play 승률 행렬을 적합도로, 진화로 수치를 탐색해 '챔피언 간 승률 분산 최소화 + 다양성 유지'를 동시 최적화하는 자동 밸런서". metric: 챔피언 pick/win rate 분산, 전략 다양성(엔트로피). IEEE CoG/ToG가 매우 선호하는 기여.
4. **진화 + gradient 하이브리드.** ES의 탐험 + RL의 효율. **Open problem:** "QD가 발견한 다양한 정책을 RL self-play의 *상대 풀(league)* 로 공급해 §1.4의 non-transitivity·전략 붕괴를 완화". §1·§3을 잇는 다리.

### 3.5 Thesis statement 예시

- "행동 특성 공간을 채우는 MAP-Elites 아카이브로 생성한 MOBA 봇 집합은, 단일 RL 정책 대비 사람이 지각하는 행동 다양성(user study)을 유의하게 높이면서 개별 봇의 강도 손실을 일정 범위 내로 유지한다."
- "self-play 승률 행렬을 적합도로 하는 QD 기반 자동 밸런서는, 수동 튜닝 대비 챔피언 승률 분산을 절반 이하로 줄이면서 전략 다양성(행동 엔트로피)을 보존한다."

### 3.6 평가 방법 (지표)

- **QD 전용 지표(필수)**: **coverage**(아카이브가 채운 칸 비율), **QD-score**(채워진 칸 fitness 총합), best fitness. 단일 best만 보면 QD의 핵심을 놓친다.
- **다양성**: 행동 디스크립터 분포, 행동 엔트로피. 사람 지각 다양성은 user study.
- **밸런싱**: 승률/픽률 분산, 메타 전략 수, Nash 분포의 support 크기.
- **PCG**: expressivity(생성 분포의 넓이), 플레이 가능성 통과율, 난이도 곡선 적합.
- **효율**: 목표 coverage/QD-score 도달까지 평가(매치) 수.
- **Ablation**: 변이/교차 연산자, archive 해상도, CMA 적응 유무.
- **Baseline**: 무작위 탐색, 순수 GA(QD 없이), CMA-ES 단일목표. 통계: 여러 seed의 QD-score 신뢰구간.
- **Threats to validity**: behavior descriptor 선택이 결과를 좌우(연구자 자유도) → 사전 등록·다중 디스크립터 검증.

### 3.7 Winters 엔진 연결점 (testbed)

- **결정론 self-play = 신뢰할 수 있는 fitness.** 진화의 적합도 노이즈가 작아진다(같은 매치업은 같은 결과 → 평가 분산 ↓). `GameRoom`에 봇 N개를 배치한 headless self-play가 fitness 평가기.
- **`GameCommand` 정책 파라미터화가 곧 genotype.** 기존 `BotLaneAIComponent`(handoff: `Shared/GameSim/Components/BotLaneAIComponent.h`)의 difficulty/scan range/leash/decisionInterval 등은 이미 *진화 가능한 파라미터 벡터*다. NEAT를 쓰면 command 선택망 자체를 진화. QD의 behavior descriptor는 `AIDebugPanel`이 이미 노출하는 state 분포·target 종류·교전 빈도에서 직접 산출.
- **자동 밸런싱 testbed.** `Shared/GameSim/Definitions/ChampionRuntimeDefaults.cpp`(쿨다운·range·lock·playbackRate 등 권위 수치)가 진화 대상 유전체. self-play 승률 행렬 → 적합도 → 수치 진화 루프를 GameSim 위에 직접 구성. 단, handoff가 경고한 "fast verification timing"이 켜져 있으면 수치가 0.2초로 덮이므로(밸런싱 평가 오염) 반드시 끄고 평가.
- **PCG testbed(오픈월드 측면).** Winters의 오픈월드 지형/스폰 데이터가 search-based PCG의 콘텐츠 공간. MOBA 라인전 외에 오픈월드까지 확장하면 기여 폭이 넓어진다.
- **§1·§2와의 다리.** QD가 만든 다양한 봇 집단은 §1 RL self-play의 상대 풀(AlphaStar league 유사)과 §2 MCTS의 시나리오 분포로 재사용 — 한 testbed가 세 분야를 동시에 지탱.

## 종합. 이 분야 통합 학위논문 구조 예시 (Three-Papers 모델)

가이드 §4의 "Three Papers Make a Thesis"를 게임 AI에 적용한다. 핵심은 §1의 "분야 목록을 다 자르지 말 것" — 대신 **하나의 일관된 명제** 아래 **인접한 세 문제**를 묶는다. 아래는 Winters의 결정론 제약을 관통선(through-line)으로 삼은 예시다.

> **통합 thesis statement(예):**
> "*결정론 server-authoritative MOBA에서, 학습(RL)·탐색(MCTS)·진화(QD)를 동일한 `GameCommand` 인터페이스 위에 통합하면, 비트 단위 재현성과 tick 단위 실시간 예산을 깨지 않으면서, 강하고(strong)·실시간이며(real-time)·다양한(diverse) 봇을 산업적으로 배포 가능한 비용으로 생성할 수 있다.*"

```text
┌─ Ch 1. 서론
│   1.1 동기: 산업 MOBA AI는 강함뿐 아니라 결정론·실시간 예산·다양성·제어가능성을 요구
│   1.2 문제: 기존 SOTA(OpenAI Five/AlphaStar)는 결정론 재현·ms 예산 제약을 다루지 않음
│   1.3 Thesis statement (위 한 문장)
│   1.4 기여: [논문1] 결정론·예산 제약 하 RL 배포 / [논문2] 실시간 search / [논문3] QD 다양성·밸런싱
│   1.5 Outline
│
├─ Ch 2. 배경 및 관련 연구
│   MDP·Bellman·PG·AC / DQN·PPO / AlphaGo·Zero·MuZero / OpenAI Five·AlphaStar
│   MCTS·UCT·PUCT·survey / GA·NEAT·ES·CMA-ES·MAP-Elites·SB-PCG
│   gap 명시: "결정론 재현 + 실시간 예산 + 다양성"을 동시에 다룬 연구 부재
│
├─ Ch 3. [논문 1] 결정론·예산 제약 하 RL 정책 배포  → 투고: NeurIPS/IEEE CoG
│   기여: 정수/결정론 추론으로 distillation한 PPO self-play 정책
│   평가: 승률·샘플효율·tick당 ms·cross-platform 비트 재현률 (§1.6)
│
├─ Ch 4. [논문 2] 결정론 GameSim을 모델로 쓰는 실시간 search  → 투고: AIIDE/IEEE CoG
│   기여: budget-aware PUCT + MOBA 연속·동시 행동공간 처리(progressive widening + SM)
│   평가: 예산 대비 품질 곡선·NN 호출 효율·결정론 (§2.6)
│
├─ Ch 5. [논문 3] QD 기반 다양성 봇 + 자동 밸런싱  → 투고: IEEE CoG / ToG
│   기여: MAP-Elites 봇 레퍼토리 + self-play 승률 적합도 자동 밸런서
│   평가: coverage·QD-score·지각 다양성 user study·승률 분산 (§3.6)
│
├─ Ch 6. 종합 평가
│   세 기여를 하나의 Winters testbed에서 통합 측정. 공통 baseline·통계·ablation.
│   "search-augmented self-play + QD 상대 풀"의 시너지 정량화.
│
├─ Ch 7. 논의 (일반화 가능성, 다른 결정론 게임으로의 전이, threats to validity)
│
└─ Ch 8. 결론 및 향후 연구 (thesis statement 증명 정리)
```

대안 묶음 두 가지(같은 분야, 다른 관통선):
- **순수 RL 박사**: [PPO 결정론 배포] + [난이도/스타일 조건화 정책] + [밸런스 패치 robust 전이] → "산업 배포 가능한 제어가능 RL 봇". 전부 §1 안.
- **순수 search 박사**: [MOBA용 SM-MCTS 수렴 분석] + [budget-aware selective NN 평가] + [결정론 비트 재현 search 시스템] → "실시간 결정론 게임의 search". §2 안 + 유형 4(증명) 강조.

핵심 원칙(가이드 §8): **각 논문을 먼저 동료심사 통과시키고, 학위논문은 마지막에 서론·결론으로 "엮는다."** 그래서 통합 thesis statement가 *처음부터* 명확해야 인접 세 문제를 고를 수 있다.
